# 마포구 구획(Block) 설계 초안 v0.1

> **작성**: Luca (게임 설계)
> **상태**: 초안 — Buddy 데이터로 검증 필요
> **기준**: 실제 마포구 지리, 역세권, 상권, 주요 도로, 랜드마크

---

## 설계 원칙

### 구획 분할 기준 (우선순위)
1. **지하철역 역세권** — 역 중심 500m 반경이 하나의 생활권
2. **주요 도로** — 양화로, 마포대로, 성산로, 백범로 등이 자연 경계
3. **용도 차이** — 상업지 vs 주거지 vs 업무지 vs 녹지가 뚜렷하면 분리
4. **랜드마크** — 시장, 대학, 공원, DMC 등 핵심 시설이 앵커
5. **생활권 인식** — 주민이 체감하는 "우리 동네" 범위

### 구획 속성 (Phase 1 데이터 모델)
```javascript
block = {
    id: string,           // "gongdeok_station_hub"
    name: string,         // "공덕역 환승허브"
    dong: string,         // "공덕동"
    type: string,         // primary: commercial|residential|mixed|office|green|industrial|transit|culture
    subtype: string,      // secondary: station|market|campus|park|redevelop|traditional
    slotCapacity: number, // 시설 배치 가능 슬롯 (2~4)
    facilities: string[], // 기존 시설
    traits: string[],     // 게임 특성 태그
    weight: {             // 동 집계 기여 비중
        livingPop: number,    // 생활인구 비중
        economy: number,      // 경제 비중
        satisfaction: number, // 만족도 비중
    },
    adjacentBlocks: string[], // 인접 구획 (동 내부 + 동 경계 넘어)
}
```

### 구획 수 가이드라인
- 소규모 동 (아현, 도화, 용강, 염리, 신수): 4~5개
- 중규모 동 (공덕, 대흥, 서강, 합정, 망원1, 망원2, 연남): 5~6개
- 대규모 동 (서교, 성산1, 성산2, 상암): 6~8개
- **총 목표: 90~110개 구획**

## Phase 1 시설 배치 자동화 규칙

Phase 1에서 플레이어가 "서교동에 공영주차장"을 선택하면:

```javascript
function autoPlaceInDong(dong, facilityType) {
    // 시설 유형에 맞는 구획 타입 우선순위
    const preference = FACILITY_PLACEMENT_RULES[facilityType];
    // 예: 공영주차장 → transit > commercial > mixed
    
    const candidates = dong.blocks
        .filter(b => b.emptySlots > 0)
        .sort((a, b) => preference.score(b.type) - preference.score(a.type));
    
    if (candidates.length === 0) throw "슬롯 부족";
    
    // 최적 구획에 자동 배치
    const target = candidates[0];
    
    // AI 자문관 코멘트: "홍대입구역 구획에 배치했습니다. 유동인구가 많아 이용률이 높겠지만, 주차장 건설로 상업 슬롯 하나를 쓰게 됩니다."
    return { block: target, tradeoff: generateTradeoff(target, facilityType) };
}
```

Phase 2에서는 플레이어가 직접 구획을 선택.
