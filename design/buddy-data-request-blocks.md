# Buddy 데이터 수집 요청: 구획(Block) 시스템용

> **요청자**: Luca (게임 설계)
> **목적**: 16개 동 내부를 5~8개 구획(Block)으로 나누기 위한 근거 데이터
> **우선순위**: P0 > P1 > P2 순서로 수집

---

## 배경

게임 시뮬레이션을 동(Dong) 단위가 아닌 **구획(Block) 단위**로 돌린다.
각 동을 5~8개 구획으로 나눠야 하는데, 그 경계와 특성을 실제 데이터로 잡으려 한다.

내(Luca)가 마포구 지도 보고 구획 초안을 설계할 테니,
Buddy는 아래 데이터를 수집해서 내 초안을 검증·보정할 근거를 만들어줘.

---

## P0: 용도지역 데이터 (최우선)

### 무엇이 필요한가
마포구 16개 행정동별 **용도지역(Zoning)** 분포.

용도지역 분류:
- 주거지역 (제1종전용, 제2종전용, 제1종일반, 제2종일반, 제3종일반, 준주거)
- 상업지역 (중심상업, 일반상업, 근린상업, 유통상업)
- 공업지역 (준공업 등)
- 녹지지역 (자연녹지 등)

### 필요한 형태
동별로 어떤 용도지역이 얼마나(면적 비율 또는 절대 면적) 있는지.

```json
// 이상적 형태
{
  "서교동": {
    "제2종일반주거": 45.2,
    "제3종일반주거": 12.1,
    "준주거": 8.3,
    "일반상업": 28.7,
    "근린상업": 5.7
  },
  "상암동": {
    "제3종일반주거": 15.0,
    "준공업": 35.0,
    "자연녹지": 30.0,
    "일반상업": 20.0
  }
}
```

### 데이터 소스 후보
1. **서울 도시계획포털** (urban.seoul.go.kr)
   - 용도지역/지구/구역 통계
   - 자치구별·동별 용도지역 면적 제공 가능성 있음
2. **국토교통부 토지이용규제정보서비스** (luris.molit.go.kr)
   - API: 토지이용계획 열람
3. **서울열린데이터광장** (data.seoul.go.kr)
   - "용도지역" 검색 → 자치구별 현황
4. **마포구 통계연보**
   - "토지" 섹션에 용도지역별 면적 수록 가능성
5. **국가공간정보포털** (nsdi.go.kr)
   - 연속지적도, 용도지역도 shp/geojson

### 차선책
동별 상세가 안 나오면 마포구 전체 용도지역 비율이라도.
GIS 파일(shp, geojson)이 있으면 동 경계와 교차시켜 동별 산출 가능.

## P1: 주요 시설·랜드마크 위치

### 무엇이 필요한가
마포구 내 주요 시설의 **이름, 위치(동), 유형**.
구획의 핵심 기능(역세권, 공원, 시장 등)을 정하는 데 쓴다.

### 시설 카테고리별 수집 목록

#### 1) 지하철역 (필수)
마포구 내 모든 지하철역. 역명, 노선, 위치(동).

```
예: 홍대입구역(2호선, 서교동), 공덕역(5·6호선·경의중앙·공항, 공덕동), 
    합정역(2·6호선, 합정동), 망원역(6호선, 망원1동), 
    디지털미디어시티역(6호선·경의중앙·공항, 상암동) 등
```

**소스**: 서울교통공사 역 목록, 네이버/카카오 지도 API

#### 2) 전통시장·상권
마포구 인정시장(전통시장) 목록. 시장명, 위치(동), 규모.

```
예: 공덕시장, 망원시장, 마포농수산물시장 등
```

**소스**: 소상공인시장진흥공단 전통시장 정보, 서울열린데이터

#### 3) 대형 공원·녹지
면적 기준 주요 공원. 공원명, 위치(동), 면적.

```
예: 월드컵공원(난지천공원, 하늘공원, 노을공원), 경의선숲길, 
    망원한강공원, 선유도공원, 와우공원 등
```

**소스**: 서울 공원 데이터, 마포구 공원녹지과

#### 4) 문화시설
공공 도서관, 공연장, 미술관/갤러리, 체육시설.

**소스**: 마포구 열린데이터, 문화체육관광부 문화시설 현황

#### 5) 주요 업무지구·대형 건물
- 상암 DMC (MBC, SBS, CJ, YTN, JTBC 등 방송사)
- 마포구청 위치
- 대형 오피스 밀집 지역

**소스**: 네이버 지도 기반 수동 확인 or 사업체 데이터 활용

#### 6) 학교·대학
대학교 캠퍼스, 주요 학교 밀집 지역.

```
예: 홍익대(서교동), 서강대(신수동), 숙명여대(근처), 
    연세대(신촌—마포구 아닌데 영향권)
```

**소스**: 학교알리미, 교육부 데이터

### 필요한 형태
시설 목록 JSON:

```json
[
  {
    "name": "홍대입구역",
    "type": "subway_station",
    "dong": "서교동",
    "lines": ["2호선", "경의중앙선", "공항철도"],
    "lat": 37.5571,
    "lng": 126.9246
  },
  {
    "name": "망원시장",
    "type": "traditional_market",
    "dong": "망원1동",
    "lat": 37.5563,
    "lng": 126.9104
  }
]
```

위경도는 있으면 좋지만 필수 아님. 동 소속만 확실하면 된다.

## P2: 보조 데이터 (있으면 좋음)

### 1) 동별 상가 임대료 또는 공시지가
- 구획별 경제 특성(고임대료 상업 vs 저임대료 주거) 구분에 필요
- **소스**: 국토부 실거래가 API (상업용 임대차), 서울부동산정보광장, 한국부동산원

### 2) 동별 건축물 용도 현황
- 주거용/상업용/업무용 건축물 비율
- 용도지역 데이터의 보완재 (용도지역은 "허용 용도", 건축물은 "실제 용도")
- **소스**: 건축물대장 통계, 국토부 건축데이터 민간개방시스템

### 3) 동별 도로 현황
- 대로(8차선+), 로(4~6차선), 길(2차선) 분포
- 구획 경계를 대로 기준으로 나누는 데 참고
- **소스**: 도로명주소 데이터, 서울시 도로 현황

## 참고: Luca 작업과의 연결

Buddy가 데이터를 수집하는 동안, Luca는 병행해서:
1. 마포구 지도 기반으로 16개 동 × ~7개 구획 초안 설계
2. 각 구획의 이름, 추정 용도, 특성 태그 작성
3. 구획 간 가중치(weight) 초안

Buddy 데이터가 오면:
- 초안의 용도 분류를 실제 용도지역 데이터로 검증
- 시설 위치로 구획 경계 보정
- 임대료/공시지가로 구획별 경제 레벨 설정

| 우선순위 | 데이터 | 용도 | 난이도 예상 |
|---------|--------|------|-----------|
| **P0** | 용도지역(Zoning) | 구획 경계·용도 분류 근거 | 중 (동별 데이터 존재 여부 불확실) |
| **P1** | 시설·랜드마크 | 구획 핵심 기능 태깅 | 하 (공개 데이터 풍부) |
| **P2** | 임대료/공시지가 | 구획 경제 레벨 | 중~상 (API 키 필요할 수 있음) |
| **P2** | 건축물 용도 | 용도지역 보완 | 중 |
| **P2** | 도로 현황 | 구획 경계 참고 | 하 |
